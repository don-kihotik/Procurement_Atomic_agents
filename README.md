# Procurement Atomic Agents

Прототипы узких закупочных AI-агентов, вынесенных из большой мультиагентной системы.
Цель — выложить их как отдельных агентов на Яндекс AI Studio.

## Принцип: одно ядро, переключаемый провайдер

```
        ┌─────────────────────────────────────────────┐
        │  Детерминированное ядро (наше, не меняется)   │
        │  should-cost · нормы спецификаций · CUAD ·    │
        │  рычаги переговоров · провенанс               │
        └───────────────┬─────────────────────────────┘
                        │  (математика и данные — провайдер-независимы)
        ┌───────────────┴─────────────────────────────┐
        │  LLM-слой (переключаемый одной переменной)    │
        │  LLM_PROVIDER = local | claude | yandex       │
        └─────────────────────────────────────────────┘
```

Ценность и устойчивость — в детерминированном ядре и проприетарных данных, а не в самой
модели. Поэтому LLM делает только извлечение/суждение, а числа считает Python — и модель
можно менять под фазу без переписывания логики.

## Три фазы (одна переменная окружения `LLM_PROVIDER`)

| Фаза | `LLM_PROVIDER` | Чем считает LLM-шаг | Зачем |
|---|---|---|---|
| 1. Локально | `local` | Ollama (gemma4) по OpenAI-совместимому эндпоинту | работает офлайн, без ключей и облака |
| 1b. Без модели | `stub` | детерминированная заглушка | прогон ядра и тестов без LLM вовсе |
| 2. На Claude | `claude` | Anthropic API (claude-sonnet-4-6) | качество рассуждения для отладки |
| 3. Яндекс | `yandex` | YandexGPT / DeepSeek по OpenAI-совместимому эндпоинту | целевая инфраструктура (РФ-контур) |

Код агента **один и тот же** во всех фазах — меняется только клиент (`procurement_agents/llm.py`).

## Три прототипа

1. **`spec`** — спецификация → оспаривание завышения (S1) + should-cost себестоимости.
2. **`contract`** — договор → разбор рисков по CUAD + правки + провенанс *(следующий шаг)*.
3. **`negotiation`** — контекст → рычаги, запасная позиция, цели, тезисы *(следующий шаг)*.

## Запуск

```bash
python -m venv .venv
. .venv/Scripts/activate                 # Windows: .venv\Scripts\activate
pip install -e .

# офлайн, без модели:
LLM_PROVIDER=stub  python cli.py spec fixtures/spec_cable.txt

# локально на Ollama:
LLM_PROVIDER=local OLLAMA_MODEL=gemma4 python cli.py spec fixtures/spec_cable.txt

# на Claude (нужен ANTHROPIC_API_KEY):
LLM_PROVIDER=claude python cli.py spec fixtures/spec_cable.txt

# на Яндексе (нужны YANDEX_API_KEY, YANDEX_FOLDER_ID):
LLM_PROVIDER=yandex python cli.py spec fixtures/spec_cable.txt
```

Скопируй `.env.example` в `.env` и заполни ключи нужной фазы.

## Структура

```
procurement_agents/
  llm.py                 # фабрика провайдера (local/claude/yandex/stub) → instructor-клиент
  core/                  # ДЕТЕРМИНИРОВАННОЕ ЯДРО (наше, провайдер-независимо)
    shouldcost.py        # статическая декомпозиция себестоимости (порт из should-cost skill)
    data.py              # BOM-шаблоны, сырьевые цены, нормы спецификаций
  schemas/               # Pydantic-контракты вход/выход
  agents/                # атомарные агенты (LLM-извлечение + вызов ядра)
cli.py                   # запуск из консоли
fixtures/                # примеры входных документов
tests/                   # тесты ядра (без LLM)
```
